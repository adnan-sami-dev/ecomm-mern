* @import "tailwindcss"; 
  > new and more efficient way to inlcude tailwind in CSS file
  > replaces older method of manually importing tailwind base, utlities and component classes in CSS

* font-optical-sizing:
  > the typographic practice of adjusting the design of letterforms
    — such as stroke thickness, character spacing, and contrast
    —based on the size at which they will be displayed. 
    It ensures that text remains highly legible at tiny sizes and elegant at massive sizes

* a common strategy to push sidebars (toggle on and off) is to use dynamic CSS:
  ${ cartOpen? translate-x-0 : translate-x-full } 
  > '0' implies moving the sidebar to it's original position
  > 'full' implies moving it across its entire width (not the window width !)

* when using useState note that we are destructuring an array
  return, not an object, so this ( { value, setValue } = useState() )
  is wrong ; this ( [value, setValue ] = useState() ) is correct.

* 'overflow-y-auto' on cart drawer content enables a scrollbar only when needed, 
  letting users scroll through items while keeping the drawer’s height fixed.

* used 'transition-colors' instead of 'transition' for slight performance advantages
  ref: checkout button on cart drawer component

* tracking-tight reduces the space between characters, 
  making text appear more compact. In Tailwind, it sets [ letter-spacing: -0.025em; ].

* & vs && operators

* can't call a state changer function inside a map method > unpredictable behaviour

* you can mark an item as null inside the map method, and then filter it out for deleting it.
  ref: <CartContents />

* to link product listings to cart content, I lifted the cartItems state up to the parent
  [ easier than contextAPI for testing the front-end ]
